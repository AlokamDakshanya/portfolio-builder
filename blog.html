Building the Website

Introduction: Building a portfolio website is a great way to showcase your projects and build a personal brand. In this blog, I’ll explain how I created a multi-page portfolio site and hosted it using GitHub Pages.

Planning

My goal was to keep it clean and simple. I designed each page with consistent structure, clear layout, and accessible navigation.

Step 1: Coding the Website

  • HTML & CSS: Used for layout, structure, and responsiveness.
  • JavaScript: Enabled interactivity and nav highlighting.
  • VS Code: My go-to editor for writing code efficiently.

Step 2: Project Showcase with Cards

I used grid-based cards on the projects page to display each project with its title, description, and GitHub link. This created a professional and visually appealing layout.

Step 3: Hosting with GitHub Pages

I pushed my code to a GitHub repo and enabled GitHub Pages via: Settings > Pages → selected main branch as source.

Final Thoughts

This experience helped me practice modular design and clean UI building. It also gave me confidence in using GitHub Pages for free, reliable hosting.
