📊 How I Built My Portfolio Website

Creating my personal portfolio website was a rewarding experience where I brought together skills in HTML, CSS, and JavaScript. My goal was to design a clean, professional site that reflects my background in data analytics and tech.

🔧 Technologies Used:
- HTML & CSS: Structured the content and styled the layout using clean and responsive design.
- JavaScript: Handled interactivity and dynamic behavior like active menu highlighting.
- Git & GitHub: Managed version control and hosted the website using GitHub Pages.
- VS Code: My development environment of choice, great for writing and managing code.

🧱 Page Structure:
- Home Page: Short intro with a call to action.
- About Me: Highlights my skills, tools, education, and work history.
- Projects: Each project is presented with a title, short description, and a file access button.
- Blog: This page! Used to document my journey, share insights, and track growth.

💡 What I Learned:
- Importance of clean UI and consistent layout.
- GitHub Pages is an easy and free way to publish your site.
- Writing modular HTML/CSS makes it easier to manage multiple pages.

🌐 Deployment:
I pushed all my files to a GitHub repo and enabled GitHub Pages under the Settings > Pages tab. The main branch with the root folder served as the source, making my portfolio live and shareable with anyone.
